# AI 聊天室 提示词

## 应用名称：AI 聊天室

## 应用概述
基于即时通信技术的局域网应用，支持本地用户实时聊天、调用第三方服务及 AI 交互，采用前后端分离架构。

## 技术架构
- 前端：Vue3 + Axios + Pinia
- 后端：Python + Flask + WebSocket + SQLite

### 环境要求
1. 后端运行于 Python 虚拟环境；
2. 需处理跨域问题，避免跨域报错；
3. 配套完整 `README.md` 文档，包含功能描述、项目结构、运行步骤。

## 第三方接口
### 1. 热门新闻接口
请求方式：`GET`
地址：`https://v2.xxapi.cn/api/baiduhot`
返回字段：新闻标题、描述、图片、热度、原链接

### 2. 随机歌曲接口
请求方式：`GET`
地址：`https://api.52vmy.cn/api/music/wy/rand`
返回字段：歌曲名称、歌手、封面、播放链接、歌曲 ID

### 3. 天气查询接口
请求方式：`GET`
地址：`https://v2.xxapi.cn/api/weather`
请求参数：
- `city`：城市名称
- `key`：`2d5c904b712686d2`
返回内容：未来 6 天天气数据

### 4. 视频播放接口
请求方式：`GET`
地址：`https://jx.playerjy.com`
请求参数：
- `url`：视频原链接
返回：可直接渲染的 HTML

### 5. AI 聊天接口
请求方式：`POST`
地址：`https://api.siliconflow.cn/v1/chat/completions`

请求 Headers：
```http
Authorization: Bearer YOUR_SILICONFLOW_API_KEY
Content-Type: application/json
```

请求体示例：
```json
{
  "model": "THUDM/GLM-4-9B-0414",
  "messages": [
    {
      "role": "user",
      "content": "用户提问内容"
    }
  ]
}
```
返回：AI 回复文本内容

## 核心功能
### 1. 登录与注册
- 访问前端默认进入登录页面，输入账号密码可跳转房间创建/加入界面；无账号可点击「立即注册」跳转注册页。
- 注册页面：填写账号、密码、选择头像，注册成功自动跳转登录页，失败停留在注册页并提示错误。
- 登录页面：账号密码校验通过进入房间列表，校验失败留在登录页提示。

### 2. 房间管理
- 房间列表页展示全部房间：房间名称、当前在线人数，每条房间附带「加入房间」按钮。
- 点击「加入房间」直接进入对应聊天界面。
- 右上角「创建房间」：输入房间名称后新建并跳转聊天界面。

### 3. 聊天界面与指令功能
#### 界面布局
- 顶部：聊天室名称、在线用户总数
- 左侧：房间成员列表（头像 + 用户名）
- 右侧上区：聊天消息展示区
- 右侧下区：消息输入框 + Emoji 表情选择器 + 发送按钮

#### 基础聊天
输入文本点击发送，房间内所有用户实时同步接收消息。

#### 指令功能（系统自动回复卡片）
1. `@新闻`
调用新闻接口，卡片展示新闻图、标题、简介，点击卡片跳转新闻原链接；系统身份输出。

2. `@听歌`
调用随机歌曲接口，音乐卡片展示歌名、歌手、封面，支持播放/暂停/下一曲；下一曲重新请求接口。

3. `@天气 城市名`
例：`@天气 南宁`，卡片展示该城市未来6天：日期、气温、天气状况、风向、空气质量。

4. `@视频 视频链接`
例：`@视频 https://xxx.mp4`，聊天区直接嵌入播放器 HTML 渲染播放。

5. `@AI 提问内容`
例：`@AI 讲解时序预测LSTM`，调用大模型接口，以AI身份回复对话内容。

### 界面适配
仅适配 PC 端，页面美观简洁。

## 数据存储
使用 SQLite 持久化存储：
- 用户信息：账号、加密密码、头像
- 房间基础信息、在线用户关联

## 异常处理
所有第三方接口请求失败时弹出友好提示，防止页面卡死、无响应。

## 安全规范
1. 用户密码加密入库，禁止明文存储；
2. 所有接口入参做基础校验，拦截非法参数请求。
